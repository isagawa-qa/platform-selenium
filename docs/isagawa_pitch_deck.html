The Problem

AI-generated tests break your architecture.

The cycle is predictable: Generate → breaks something → fix → generates it wrong again. You spend more time fixing AI output than writing tests yourself.

  • Tests break existing architecture patterns with every generation
  • Page Objects get skipped — locators end up in Tasks or Roles
  • Business logic mixes with UI interaction in the same function
  • The same mistakes repeat across every AI session
  • No enforcement means every new developer reintroduces the same bugs
  • CI pipelines fail hours after the AI said it was done
The Solution

The Isagawa Kernel
manages AI from within.

It doesn't monitor the AI from outside. It runs inside the agent — enforcing every action at the moment of execution, permanently learning after every failure.

⚡

Real-Time Enforcement

Hooks block writes the instant a layer violation is detected. The AI cannot proceed until the architecture is correct.

🧠

Permanent Learning

Every failure triggers a mandatory learn cycle. Lessons are recorded as hard rules. The same mistake cannot recur.

🔒

Human-in-the-Loop

On failure, the AI stops, reports, and waits. It never autonomously loops through fix attempts. You decide. It executes.

Architecture

5 Layers. One job each. No mixing.

Test
Arrange / Act / Assert — orchestrates Roles, asserts via POM state-check methods

@autologger("Task")
