## 8. Competitive Positioning Summary

### Four Quadrant Framework

- **Proprietary platforms** (mabl, Virtuoso): enforce standards but lock you in
- **AI generators** (ChatGPT, Copilot): fast but messy, no standards
- **Open source** (Serenity BDD): standards but no AI
- **Isagawa**: AI speed + Open source + Enforced standards (unique position)

### Key Distinction — AI Governance vs AI Execution Management

| AI Governance (Others) | AI Execution Management (Isagawa) |
|------------------------|-----------------------------------|
| Monitors AI behavior | Controls AI behavior |
| Documents compliance | Enforces compliance |
| Alerts on violations | Prevents violations |
| Audits after execution | Gates during execution |
| "Did the AI do it right?" | "The AI can only do it right" |

### Key Distinction — MCP-Based vs AI-Native

| MCP-Based (Others) | AI-Native (Isagawa) |
|---------------------|---------------------|
| Build coded tools | Teach the agent |
| Infrastructure that constrains | Knowledge that guides |
| Engineering effort per vertical | One markdown file per vertical |
| Linear scaling | Zero marginal cost |

## Summary

This services-first GTM strategy leverages the Trojan Horse Demo Model to:

1. Minimize friction (free demo on client's actual site)
2. Demonstrate immediate value (working code in 60 minutes)
3. Create commitment through value delivery (they keep the code)
4. Enable rapid scaling (AI does 85%, humans fine-tune 15%)

The phased targeting approach ensures we build case studies and refine the model with individual practitioners before scaling to teams and enterprises. The competitive positioning around AI Execution Management (not just governance) and AI-Native architecture (not just MCP tools) creates a defensible market position.
